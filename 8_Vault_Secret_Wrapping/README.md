# Vault Secret Wrapping Demo

This demo shows Vault response wrapping with a browser-based receiver experience.

The sender creates a single-use wrapping token for a Vault response. The recipient opens the frontend, pastes that wrapping token, and the browser calls Vault `sys/wrapping/unwrap` directly to retrieve the secret payload.

## What Terraform Creates

- A KV v2 mount at `secret-wrapping`.
- A demo secret at `secret-wrapping/handoff`.
- A `secret-wrapping-sender` policy that can read the demo secret.
- Optional Vault CORS configuration for `http://localhost:8788`.
- A local demo server that serves the frontend and proxies unwrap requests to Vault when CORS is not available.

## Deploy

```sh
terraform init
terraform apply -var-file="terraform.tfvars.example"
```

The module reads the Vault address and admin token from `../1_Create_HCP_Vault_Cluster/terraform.tfstate`.

## Run the Receiver Frontend

```sh
./scripts/serve.sh
```

The script defaults `VAULT_SKIP_VERIFY=true` for local demos where Python does not have access to the macOS certificate trust store. Set `VAULT_SKIP_VERIFY=false` before running it if your Python trust store validates the Vault certificate correctly.

Open `http://localhost:8788` and enter:

| Field | Value |
|---|---|
| Vault address | Terraform output `vault_addr` |
| Vault namespace | Terraform output `vault_namespace`, default `admin` |
| Wrapping token | Token generated by the sender command below |

## Create a Wrapping Token

Use an operator token or a token with the `secret-wrapping-sender` policy, then run:

```sh
export VAULT_ADDR="$(terraform output -raw vault_addr)"
export VAULT_NAMESPACE="$(terraform output -raw vault_namespace)"

vault kv get -wrap-ttl=5m -format=json secret-wrapping/handoff \
  | jq -r '.wrap_info.token'
```

Send only that wrapping token to the recipient. The token expires after the wrapping TTL and can be unwrapped only once.

## Notes

- The receiver never needs a normal Vault token.
- The wrapping token is a short-lived cubbyhole token.
- HCP Vault Dedicated may reject `sys/config/cors` updates from namespace-scoped admin tokens. The included local server avoids that browser limitation by proxying only the unwrap request.
- Opening `app/index.html` directly from the filesystem will not work because the frontend expects the local `/unwrap` endpoint from `./scripts/serve.sh`.
